# Как работать с изменениями без терминала (через сайт GitHub)

<img width="1200" height="475" alt="GHBanner" src="https://github.com/user-attachments/assets/0aa67016-6eaf-458a-adb2-6e31a0763ed6" />
Если ты редактируешь проект прямо на GitHub, делай так:

  <h1>Built with AI Studio</h2>
1. Открой нужный файл в репозитории.
2. Нажми ✏️ **Edit this file**.
3. Внеси изменения.
4. Нажми **Commit changes...**.
5. Заполни короткое сообщение коммита (например: `feat: add /help command`).
6. Снова нажми **Commit changes**.

  <p>The fastest path from prompt to production with Gemini.</p>
## Простыми словами

  <a href="https://aistudio.google.com/apps">Start building</a>
- **Commit** — сохранить изменения в истории проекта.
- **Push** — отправить изменения в удалённый репозиторий.

</div>
На сайте GitHub при редактировании файла ты обычно делаешь commit сразу в удалённый репозиторий, поэтому отдельный `push` руками не нужен.

## Что нужно сделать дальше по твоему плану Bible Bot

Приоритет:

1. Этап 1: техническая стабильность (system/user роли, фоновые потоки, cooldown, удаление keep_alive.py).
2. Этап 3: UX (/help, темы, кнопки после разбора).
3. Этап 4: избранное (JSON-хранилище).
4. Этап 2: расширение базы стихов.

## Мини-чек после каждого изменения

- Открывается ли бот без ошибок?
- Работает ли `/start` и `/help`?
- Работает ли разбор стиха?
- Нет ли ошибок в логах?

Если хочешь, можно продолжить в формате: «я делаю всё за тебя по шагам, ты только вставляешь полный файл и жмёшь Commit». 

## Проверка тем и ссылок

Если появится ошибка, что стих из темы не найден, пройди мини-аудит: `BIBLE_DATA_AUDIT.md`.
